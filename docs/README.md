# 公開文件

這裡放公眾可讀的「個人知識與品味脈絡系統」產品文件。公開文件必須遵守 [`../PUBLICATION_BOUNDARY.md`](../PUBLICATION_BOUNDARY.md)，並依照 [`公開文案語氣規格.md`](公開文案語氣規格.md) 撰寫。

目前 README 只承擔產品定位、公開狀態與邊界說明。正式安裝、升級、命令列入口、套件 或部署文件，必須等產品自身入口明確後再新增，不得把底層協作工具的安裝頁當作產品安裝文件。

## 寫作邊界

公開文件可以描述產品用途、安裝、升級、操作流程、示例和限制；不可搬入營運工作區的 `dev/`、`outputs/`、交接歷史、內部研發證據或私人資料。

可以簡短提及底層人工智能協作機制，但文件主體必須服務個人知識與品味脈絡系統的使用者。

公開文件的主調應由使用者能明白的產品價值開始：把看到的內容通過 AI 快速內化，保存可追溯來源，拆出知識點，生成資訊圖，並逐步形成知識脈絡與品味脈絡。
