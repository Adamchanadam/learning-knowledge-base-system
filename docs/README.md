# 公開文件

這裡放公眾可讀的產品文件。公開文件必須遵守 [`../PUBLICATION_BOUNDARY.md`](../PUBLICATION_BOUNDARY.md)。

目前安裝、升級與 AI agent 入口的單一公開真源是 [`../README.md`](../README.md)。release notes 的固定寫法記錄在 [`../PUBLICATION_BOUNDARY.md`](../PUBLICATION_BOUNDARY.md)，內容必須面向非開發者用戶。不要為基本安裝／升級原則過早拆出獨立文件；只有當產品已有多平台流程、完整 npm package 行為或 README 明顯過長時，才新增專門文件。

## 寫作邊界

公開文件可以描述產品用途、安裝、升級、操作流程和限制；不可搬入 OPS 的 `dev/`、`outputs/`、交接歷史、內部研發證據或私人資料。
