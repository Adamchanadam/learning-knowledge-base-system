# 個人知識與品味脈絡系統

個人知識與品味脈絡系統是一套由人工智能代理協助運作的本機知識工作流。它處理的不是「把網頁存起來」這件事，而是把一條來源轉成可追溯的知識真源、可回用的知識點、可覆核的個人判斷，以及日後能累積成品味脈絡的材料。

它的核心問題是：讀了很多文章、文件、貼文和長文後，怎樣避免只得到一堆摘要？怎樣把每次閱讀變成可查、可引用、可連接、可判斷、可行動的個人知識資產？

## 產品定位

這個系統的第一階段是實用的人工智能第二大腦：每條公開來源都要先經過可讀性稽核，再整理成 Obsidian Markdown 真源筆記、知識點筆記、附件、索引與檢視表。它要令使用者更快理解內容，日後也能回到原文證據，而不是只相信一次性的人工智能摘要。

第二階段是在累積足夠來源、知識點、採納與拒絕紀錄後，形成個人判斷與品味探索。這裡的「品味」不是表面喜好，而是認知、經驗、價值與行動選擇長期沉澱出的偏好系統。系統要幫使用者分辨：這條內容是補足知識缺口、加深某個概念、連接兩個原本分散的問題、修正舊判斷，還是足以推動一次真實行動。

## 一條來源進來後會發生什麼

1. 先確認來源是否公開可讀、正文是否完整；讀不到就停下，不憑標題或印象補內容。
2. 判斷來源用途：事實真源、工作流程、學習材料、判斷材料、品味材料、行動材料，或只值得暫存。
3. 生成 Obsidian 來源筆記，保存來源稽核、原文層、摘要、概念拆解、知識點、關係與學習檢查。
4. 抽取原子知識點，寫成可回連原文證據的知識點筆記。
5. 如內容與使用者的項目、判斷、生活或創作有關，另寫應用筆記；個人化判斷不污染原文真源。
6. 檢查跨來源疊加價值：補底、加深、連接、修正、行動。沒有成立就明確標記，不硬湊洞見。
7. 需要時生成面向閱讀的成果頁，幫使用者做更清楚的理解、取捨、收藏、深讀、拒絕或行動。

## 這個系統重視什麼

- 真源可追溯：每個摘要、知識點和判斷都要能回到來源內容。
- 分層清楚：原文、人工智能轉化、個人應用和處理狀態不能混在一起。
- 本機優先：正式資料層以 Obsidian 知識庫、Markdown、屬性、內部連結、附件和 Bases 檢視表承載。
- 不假裝個人化：讀不到既有脈絡時，要標記缺少脈絡，而不是強行說「這適合你」。
- 不被平台演算法牽著走：品味探索要辨認平台熱門、商業推送、同溫層重複與真正擴闊視野的內容。
- 不以數量冒充進步：新內容要說明它如何令既有知識網更有用；普通整理不能冒充洞見。

## 適合的使用者

- 想把閱讀、研究、工作資料和生活觀察沉澱成長期可用知識的人。
- 想用人工智能整理來源，但仍要求證據、分層和可覆核判斷的人。
- 想把知識連到決策、項目、生活選擇、創作方向與品味經營的人。
- 想建立本地優先、可逐步封裝成工具的個人知識工作流的人。

## 與 Obsidian 的關係

Obsidian 是本系統第一階段的正式資料層。來源筆記保存來源正文與忠實轉化；知識點筆記保存原子知識點；應用筆記保存個人化扣連、採納判斷、跨來源洞見與品味材料；Bases 檢視表只作入口與狀態查看，不取代 Markdown 真源。

這個設計的目標不是展示一個漂亮資料夾，而是讓知識能長期累積、可重跑、可去重、可讀回、可被人工智能代理按證據使用。

## 不是什麼

- 不是普通書籤工具。
- 不是一次性摘要產生器。
- 不是把所有內容都硬套成品味分析的推薦系統。
- 不是用圖片或檢視表取代正文真源的展示工具。
- 不是承諾能自動讀取所有社群平台或付費內容的爬蟲。
- 不是 Agent Handoff Kit 的安裝教學。

## 底層協作機制

本倉庫使用 Agent Handoff Kit 作為人工智能協作、跨對話接續與治理狀態保存的底層機制。它不是本產品的功能主體，也不是使用者要安裝或理解的第一入口。

對使用者而言，產品主體始終是「個人知識與品味脈絡系統」：把來源變成 Obsidian 真源、知識點、個人判斷、跨來源洞見、品味材料與可閱讀成果。
