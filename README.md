# 個人知識與品味脈絡系統

這是一個幫你把你看到的內容，通過 AI 幫你快速內化，變成自己能用、有啟發性的知識的工具。

你給它一篇文章、一個 URL 或一段文字，它不單只是幫你寫摘要，而是會保存可追溯的來源，整理重點，拆出知識點，生成幫你快速看懂原文的資訊圖，再判斷這內容跟你的工作、學習、生活選擇或審美取向有什麼關係。

時間久了，它會慢慢形成一套屬於你的知識脈絡和品味脈絡，讓你不是被平台推薦牽著走，而是更清楚自己為什麼採納、深讀、甚至拒絕某些內容。

## 它真正做的事

這個系統不是把網頁收進資料夾，也不是把長文壓成幾句摘要。它要做的是把每次閱讀、觀看和收藏，變成日後仍然有用的知識資產。

一條內容進來後，系統會先確認來源是否可讀，再把原文、重點、知識點、資訊圖、個人判斷和後續行動分開處理。這樣做的原因很簡單：原文要可追溯，知識點要可重用，個人判斷要可覆核，品味線索要慢慢累積。

## 一條內容會怎樣被處理

1. 先確認來源是否公開可讀、正文是否完整；讀不到就停下，不憑標題或印象補內容。
2. 整理原文重點，保留可回看的來源和證據。
3. 拆出能獨立使用的知識點，方便日後搜尋、引用和連接。
4. 生成資訊圖，幫你快速看懂原文的概念、流程、關係、差異或重點。
5. 判斷這條內容屬於事實資料、工作流程、學習材料、判斷材料、品味材料，還是只值得暫存。
6. 如果內容跟你的項目、生活、創作或選擇有關，另寫個人應用和採納判斷，不把個人想法混進原文。
7. 檢查它跟既有內容能否產生新價值：補足缺口、加深理解、連接問題、修正舊判斷，或推動一次行動。

## 為什麼需要資訊圖

很多內容不是「讀不讀」的問題，而是讀完後仍然抓不到結構。資訊圖的作用是把原文裡最難吸收的部分變成一眼能理解的視覺整理：一個概念怎樣定義，一個流程怎樣發生，幾個觀點有什麼差異，哪些人物、地方、工具或數字支撐了原文。

資訊圖不是裝飾，也不是給 AI 讀的資料來源。它是給人快速理解原文的交付物。個人判斷、品味分析和系統內部狀態不會畫進資訊圖，避免把原文和個人推論混在一起。

## 它重視什麼

- 可追溯：每個摘要、知識點和判斷都要能回到來源。
- 可理解：複雜內容要能用文字和資訊圖快速看懂。
- 可累積：每條內容都要能進入長期知識脈絡，而不是變成一次性摘要。
- 可分層：原文、AI 整理、個人應用和處理狀態不能混在一起。
- 可判斷：不是所有內容都值得收藏、深讀或行動；拒絕和暫存也要有理由。
- 可形成品味：長期來看，系統要幫你看清自己重視什麼、抗拒什麼、值得探索什麼。

## 與 Obsidian 的關係

Obsidian 是目前的主要保存方式。來源筆記保存原文和整理後的知識內容；知識點筆記保存可重用的概念和主張；應用筆記保存個人判斷、採納理由、跨來源洞見和品味線索；資訊圖和其他素材則保存為可回看的附件。

這樣設計，是為了讓資料留在本機，長期可讀、可查、可重跑、可整理，也方便日後由 AI 按證據繼續使用，而不是每次都重新理解一次。

## 適合誰

- 想把閱讀、研究、工作資料和生活觀察沉澱成長期知識的人。
- 想用 AI 加快整理，但不想失去來源和判斷依據的人。
- 想把知識連到工作決策、學習方向、生活選擇、創作和審美的人。
- 想慢慢建立自己知識脈絡和品味脈絡的人。

## 不是什麼

- 不是普通書籤工具。
- 不是一次性摘要產生器。
- 不是把所有內容都硬套成品味分析的推薦系統。
- 不是用資訊圖、圖片或表格取代原文真源的展示工具。
- 不是承諾能自動讀取所有社群平台或付費內容的爬蟲。
- 不是 Agent Handoff Kit 的安裝教學。

## 底層協作機制

本倉庫使用 Agent Handoff Kit 作為開發和維護時的底層協作機制。它不是本產品的功能主體，也不是使用者要安裝或理解的第一入口。

對使用者而言，產品主體始終是「個人知識與品味脈絡系統」：把內容變成來源真源、知識點、資訊圖、個人判斷、跨來源洞見和品味脈絡。
